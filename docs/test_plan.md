# 山竹小姐の学士帽 — 测试计划 & 测试用例

| 项目 | 说明 |
|------|------|
| 被测系统 | Flask 学士帽租赁平台（cap_shop） |
| 测试类型 | 单元测试 · API 接口测试 · E2E 浏览器自动化 |
| 时间基准 | 北京时间（Asia/Shanghai） |
| 自动化框架 | pytest · pytest-cov · Playwright |

---

## 1. 测试范围

### 1.1 在范围内

| 模块 | 说明 |
|------|------|
| 租赁计价 | 自然日计费、押金计算 |
| 库存算法 | 租期重叠、购物车叠加、取消订单释放 |
| REST API | `/api/time`、`/api/rental-quote`、`/api/inventory` |
| 预订流程 | 加入购物车、保存订单、购物车结算 |
| 管理端 | 店主登录、订单状态变更 |
| 校验逻辑 | 日期、数量 1～10、手机/地址必填 |

### 1.2 不在范围内（本期）

- 邮件 SMTP 真实发送
- 微信支付/收款码真实扣款
- 多用户并发写 `store.json` 压测
- 移动端真机兼容（仅 Playwright 模拟视口）

## 2. 测试策略

```
         ┌─────────────┐
         │  E2E (P0)   │  3 条主流程，Playwright
         └──────┬──────┘
                │
         ┌──────▼──────┐
         │ API (P0/P1) │  JSON 接口，pytest + test_client
         └──────┬──────┘
                │
         ┌──────▼──────┐
         │ 单元 (P0)   │  纯函数，pytest
         └─────────────┘
```

| 层级 | 工具 | 目录 | 触发时机 |
|------|------|------|----------|
| 单元 + API | pytest | `tests/test_*.py` | 每次 push / PR |
| E2E | Playwright | `tests/e2e/` | 每次 push / PR（CI 独立 job） |
| 覆盖率 | pytest-cov | `app.py` | 单元 job，阈值 ≥ 35% |

## 3. 测试环境

| 项 | 要求 |
|----|------|
| Python | 3.11+ |
| 依赖 | `pip install -r requirements-dev.txt` |
| E2E 浏览器 | `playwright install chromium` |
| 数据隔离 | 测试使用临时 `store.json`，不修改生产数据 |
| 固定日期 | 测试中将「今天」固定为 `2026-06-07` |

### 本地执行

```powershell
# 单元 + API（默认，不含 E2E）
py -3 -m pytest

# 仅 API
py -3 -m pytest tests/test_api.py -v

# E2E（需先安装浏览器）
playwright install chromium
py -3 -m pytest tests/e2e -v

# 覆盖率 HTML 报告
py -3 -m pytest --cov=app --cov-report=html
```

## 4. 功能测试用例

优先级：**P0** 必测 · **P1** 重要 · **P2** 一般

### 4.1 商品与浏览

| ID | 优先级 | 用例名称 | 前置条件 | 步骤 | 预期结果 |
|----|--------|----------|----------|------|----------|
| TC-001 | P1 | 点单页加载 | 服务已启动 | 访问 `/shop` | 显示商品分类与招牌区 |
| TC-002 | P1 | 商品详情 | 有效商品 ID | 访问 `/product/r08` | 显示名称、价格、库存、「填写预定」按钮 |
| TC-003 | P1 | 无效商品 | — | 访问 `/product/invalid` | 提示未找到，跳转点单页 |

### 4.2 租赁预订

| ID | 优先级 | 用例名称 | 前置条件 | 步骤 | 预期结果 |
|----|--------|----------|----------|------|----------|
| TC-010 | P0 | 加入购物车 | 库存充足 | 预订页选日期 → 点「加入购物车」 | 跳转购物车，tab 显示数量角标 |
| TC-011 | P0 | 保存订单 | 库存充足 | 填手机/地址/日期 → 保存订单 → 确认 | 跳转订单页，标题「订单已保存」 |
| TC-012 | P0 | 结束早于开始 | — | API：`start > end` | 400，提示日期无效 |
| TC-013 | P0 | 开始早于今天 | — | 选过去日期提交 | 拒绝，提示不能早于今天 |
| TC-014 | P0 | 数量边界 | — | quantity=0 / 11 | 拒绝，提示 1～10 |
| TC-015 | P0 | 库存不足 | 该时段已订满 | 再下单同日期 | 提示库存不足 |
| TC-016 | P1 | 租期计价 | — | 6/10～6/12，1 顶 | 租金 45 元（3 天×15），押金 30 |

### 4.3 购物车与结算

| ID | 优先级 | 用例名称 | 前置条件 | 步骤 | 预期结果 |
|----|--------|----------|----------|------|----------|
| TC-020 | P0 | 空车结算 | 购物车为空 | 访问 `/cart` | 显示「购物车是空的」 |
| TC-021 | P0 | 购物车结算 | 车中有商品 | 填手机/地址 → 提交全部订单 | 跳转 `/cart/done`，显示订单号与合计 |
| TC-022 | P1 | 移除商品 | 车中有商品 | 点「移除」 | 商品消失，合计更新 |
| TC-023 | P0 | 缺手机/地址 | 车中有商品 | 不填手机提交 | 提示请填写，停留购物车 |

### 4.4 管理端
